Including Bootstrap Bootstrap can be included into your page in 3 ways: It can be loaded straight from Bootstrap CDN: It can be downloaded, put into your project and linked simply with a relative URL: It can be loaded as a module using NPM and then imported into your own SCSS files. This is also the recommended way of modifying it, extending it, optimizing it and creating your own themes. @import "../../node_modules/bootstrap/scss/bootstrap"; Bootstrap Contents What does actually Bootstrap consist of? Reboot - a set of rule that standardizes element behaviour across browsers. Including this makes sure that styling in all browsers starts from the same point. Grid - the layout part of Bootstrap, allowing to create simple, grid-based, responsive page layouts. Components - various components providing useful styling to common website elements such as buttons, forms, lists, navigation, tooltips etc. Utilities - utility classes that you can apply to almost any element, removing the need to write any CSS. These include styles for borders, colors, display, floating, positioning, spacing, alignment and many others. Bootstrap javascript To provide functionality to some of its components, Bootstrap also needs some JavaScript libraries. Those are built upon the jQuery library so it needs to be included as well. Furthermore, Bootstrap popups are utilizing the Popper.js library so if you plan to use them, that library needs to be included as well. You can find links to download or include all of these files on Bootstrap's download page: https://getbootstrap.com/docs/4.3/getting-started/download/ Note: these scripts need to be included before bootstrap. Bootstrap versions Bootstrap is always evolving and new versions are not always backwards compatible with the old ones. Always when working with Bootstrap check the version of the documentation you are using as reference. The current version of Bootstrap (September 2019) is 4.3 Bootstrap requirements Bootstrap 4 should run flawlessly on any modern browser including IE11 and Edge.
